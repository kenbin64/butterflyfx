bfx-ingest

A zero-dependency CLI that turns a codebase into clean, deterministic LLM context in one command. On a real 1,593-file repo it produced one verifiable artifact reported at ~4.55M tokens, so you scope before wasting a call. Same input gives the same root hash, which makes context reproducible for evals and byte-identical for prompt caching, cutting repeated input cost by roughly 85% in a typical session.

How I work, and where it is going

I get this reliability out of stock models by directing them, not hoping. Explicit operating directives, hard constraints, and a verification pass at each step keep a model on track and out of drift. Reduced to a single number, that discipline took a verifier's hallucinated output from about 39% to 0% while keeping the answer rate high. The point is not a smarter model, which understandably makes people wary; it is a cheaper and more reliable one. I focus my instance of it; the provider's model itself is untouched.

Since August 2025, on my own initiative, I have gone past using AI into how it works and how to optimize it, and I am developing an original framework for it: dimensional programming, representing data as derivable geometry so a model reads only what it needs instead of being handed everything. It is demonstrated, not just claimed. A dependency-free API measures a roughly 99.7% token reduction answering localized questions over a large nested structure. The ideas are mine; every claim is published with a clear label for how well it is supported, so the bold parts stay bold and the whole thing stays honest.

Cost and latency

Reproducibility & provenance
