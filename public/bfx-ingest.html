bfx-ingest
Deterministic LLM context from any codebase · in one command
Feeding a repo to an AI is messy: you paste files by hand, blow the context window, and get a different blob every time so you can't cache or reproduce it. bfx-ingest turns a whole codebase into one clean, token-counted, verifiable context artifact. Open source. No dependencies. No network. Your code never leaves your machine.

The problem, and the fix

Working with an LLM on real code means assembling context. Done by hand it is slow, incomplete, and non-reproducible: you forget files, send the same vendored code twice, overflow the window, and can never recreate the exact input you used yesterday. That breaks evals, wastes tokens, and misses the prompt cache.

bfx-ingest walks the folder, content-addresses every file, dedupes identical ones, and emits one artifact in the shape your model wants (Markdown, XML, or JSON), with a token count and a reproducible root hash. Same repo in, same bytes and same hash out, on any machine.

Install and use

One command. Nothing to set up.

# the whole repo into Markdown context for GPT/Cursor npx bfx-ingest ./my-project --format md --out context.md # Claude prefers XML; pipe straight to your clipboard npx bfx-ingest ./src --format xml | clip # JSON for your own RAG / eval pipeline npx bfx-ingest ./src --format json > ctx.json

Options: --format md|xml|json, --out FILE, --max-kb N. It skips the usual junk (.git, node_modules, build output, binaries, lockfiles) and honours a basic .gitignore.

Why you can trust it

No black box

The whole tool is one small, readable file of plain Node. No build step, nothing compiled or obfuscated, MIT licensed. Read it before you run it.

Local and offline

Pure Node, zero dependencies, no network. Your code never leaves your machine.

Verify, don't trust

Same repo in gives the same root hash out, on any machine. Re-run it and check, instead of taking my word.

Reproducible by design

The hash and manifest prove exactly what the model saw, for evals, prompt caching, and provenance.
